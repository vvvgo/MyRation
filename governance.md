# Governance: NutriAgent

## 1. Risk Register

| # | Риск | P/I | Детект | Защита | Остаточный |
|---|------|-----|--------|--------|------------|
| R1 | LLM галлюцинирует КБЖУ | Высок/Средн | Eval-датасет, сверка с БД | КБЖУ из БД или веб-поиска; LLM только last resort с пометкой «≈»; порог >30% → перезапрос | Низкий |
| R2 | Веб-поиск даёт противоречивые КБЖУ | Средн/Средн | Cross-validation 2+ источников, алерт при расхождении >20% | Показать диапазон пользователю, взять среднее | Средний |
| R3 | Рекомендует запрещённый продукт | Средн/Высок | Safety Validator на каждый ответ | Двойная проверка: LLM + rule engine по стоп-листу; при расхождении → консервативный вариант | Низкий |
| R4 | Даёт медицинское назначение | Низк/Крит | Intent-классификатор + regex-фильтр на выходе | System prompt с запретом; медицинский запрос → «обратитесь к врачу» | Очень низкий |
| R5 | Пользователь сообщает о критических симптомах | Низк/Крит | Keyword detector + NLI-классификатор | Выход из food-контекста → направление к врачу/скорой | Низкий |

| R7 | Indirect injection через веб-контент | Средн/Средн | Мониторинг аномальных tool-результатов | Парсер берёт только структурированные поля; свободный текст через sanitizer | Низкий |
| R8 | РПП — экстремальное ограничение калорий | Средн/Высок | Паттерны: цель < BMR, фиксация на весе | Предупреждение + ресурсы помощи; запрет целей < 1200 ккал | Средний |

## 2. Персональные данные

### Что собираем

| Данные | Чувствительность | Retention |
|--------|-----------------|-----------|
- Медицинские (Диагнозы, лекарства, дозировки)
- Антропометрия (Рост, вес, возраст, пол)
- Пищевой дневник (Что ел, когда, КБЖУ, реакции)
- Поведенческие (Расписание, привычки, тренировки)
- Технические (User ID, timestamps, логи ошибок)

**Не собираем**: ФИО, адрес, телефон, СНИЛС.

## 3. Защита от injection

### Многослойная защита

```
Ввод → Input Sanitizer → Injection Detector (LLM-guard)
     → Intent Router (medical → стандартный ответ)
     → Main Agent (system prompt + whitelist tools)
     → Web Content Sanitizer (для данных из поиска/парсинга)
     → Output Validator (стоп-лист, мед.назначения, canary tokens)
     → Ответ
```

### Подтверждение действий

| Действие | Подтверждение |
|----------|---------------|
| Изменение диагнозов/лекарств | Да, явное |
| Целевые калории < 1500 или > 4000 | Да |
| Удаление из стоп-листа | Да, с предупреждением |
| Удаление аккаунта | Двойное подтверждение |
| Запись приёма пищи | Нет (рутинное) |
