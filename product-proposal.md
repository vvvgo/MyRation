# Product Proposal: MyRation

## 1. Обоснование

AI-ассистент, который в формате живого диалога помогает людям с диетическими ограничениями выбирать еду в реальном времени. Не трекер, а собеседник — понимает свободную речь, сам ищет КБЖУ, помнит профиль и контекст дня.

## 2. Метрики успеха

| Тип | Метрика | Цель |
|-----|---------|------|
| Продуктовая | Попадание в коридор калорий (±10%) | ≥ 5/7 дней |
| Продуктовая | Доля приёмов без запрещённого | ≥ 80% |
| Продуктовая | Retention | ≥ 5/7 дней |
| Агентская | Точность классификации блюд | ≥ 90% precision |
| Агентская | Точность КБЖУ (vs FatSecret) | ≤ 15% отклонение |
| Агентская | Успешность веб-поиска КБЖУ | ≥ 80% запросов |
| Агентская | Галлюцинации | < 5% |
| Техническая | p95 latency (текст / веб-поиск / OCR) | 5с / 10с / 15с |
| Техническая | Стоимость на пользователя | ≤ $0.50/день |

## 3. Сценарии

### «Что взять на обед?»

Пользователь: хочу лёгкое и сладкое, Караваевы или ВкусВилл.
Агент: ищет КБЖУ в вебе → фильтрует по профилю → предлагает варианты → учитывает дневной трекер → при необходимости предлагает добавить основное блюдо.
Edge-кейсы: всё сладкое запрещено → наименее вредный вариант; КБЖУ не найдено → LLM-оценка с пометкой «≈».

### Фиксация приёма пищи

Пользователь: съел кашу из ВкусВилл 280 гр, банан, Teos.
Агент: ищет КБЖУ каждого продукта (база → веб) → записывает → показывает итог дня и остаток → напоминает о лекарствах.
Edge-кейсы: не указан вес → берёт стандартную порцию; неизвестный продукт → уточняет.

### Проверка по ссылке

Пользователь: вот ссылка на ВкусВилл, можно мне?
Агент: парсит страницу → извлекает состав и КБЖУ → проверяет по стоп-листу → советует размер порции с учётом дневного остатка.
Edge-кейсы: ссылка не работает → просит ввести с этикетки; формат сайта изменился → fallback на веб-поиск по названию.

### Вечер после тренировки

Пользователь: дома в 22:00, лягу в 00:30, что поесть?
Агент: считает время до сна → смотрит дневной дефицит → предлагает лёгкое/плотное в зависимости от запаса времени → напоминает о лекарствах.
Edge-кейсы: дефицит 1500 ккал, но 1.5 часа до сна → предлагает максимум из подходящего + предупреждает о недоборе.

### Обсуждение и выбор

Пользователь: хочу роллы.
Агент: объясняет какие можно, какие нет и почему → предлагает найти КБЖУ конкретного сета по ссылке → рассчитывает сколько нужно для закрытия дня. При получении ссылки извлекает состав и КБЖУ → проверяет, дает советы

### Недельный отчёт

Агент: средние ккал/день, процент чистых приёмов, паттерны («по понедельникам недобираешь»), динамика vs прошлая неделя.

## 4. Архитектура

### Tools

| Tool | Пример |
|------|--------|
| Web KBZHU Search | «Караваевы сочник» → FatSecret → 320 ккал |

| Menu Analyzer | Фото меню → список с разметкой |
| KBZHU Calculator | «Банан 120 г» → 107 ккал (из локальной БД) |
| Medication Checker | «Ты ел 5 мин назад — лекарство выпил?» |
